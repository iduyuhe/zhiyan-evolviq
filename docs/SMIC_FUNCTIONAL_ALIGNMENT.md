## 二、逐项功能对齐矩阵

### 2.1 物料齐套管理 (对齐度 85%)

| SMIC业务需求 | 智衍当前状态 | 差距 | 优先级 |
|:------------|:-----------|:----|:-----:|
| 单BOM齐套检查 | ✅ Agent Studio → 规划 → 执行 | — | P0 |
| 多BOM并行检查 | ❌ 每次只能检查一个BOM | 缺少"批量作业"概念 | P1 |
| 按品类/产线筛选 | ⭕ 通过目标语言描述 | 自然语言可覆盖但不结构化 | P1 |
| 周期自动检查 | ⭕ Agent目标可设"每2小时" | 缺少定时任务调度器 | P0 |
| 检查结果推送 | ❌ 需要人打开Studio查看 | 缺少通知/推送机制 | P0 |
| 历史趋势对比 | ❌ 每次是独立检查 | 缺少前后对比能力 | P2 |

**P0级缺口（演示前必须补）**：
```
缺：定时任务调度器 + 结果推送
→ 演示时可以说"Agent可设每2小时自动检查"
但当前系统还没有定时器
```

## 三、功能缺口优先级分析

### P0级（演示必须有的）

```
优先级    功能缺口          现状                      补救措施
──────    ──────           ────                      ──────
P0-1  定时自动检查       Agent设了"每2小时"但实际没定时器 → 加一个最小定时任务（用APScheduler）
P0-2  缺料推送          结果只在Studio里显示           → 加一个"通知面板"或者简单的事件列表
P0-3  国产/进口标签     替代推荐没有区分国产进口        → 在替代数据中增加source字段
P0-4  多点一次           整个流程需要3次点击            → 加"一键快速检查"跳过规划确认（演示用）
```

### P1级（V1必须补的）

```
P1-1  多BOM批量检查         → 支持一次检查多个BOM
P1-2  供应商管理界面         → 供应商清单+评级展示
P1-3  安全库存策略设置       → Agent能理解"安全水位"
P1-4  SAP/ERP适配器         → 对接SMIC现有系统
P1-5  KPI仪表盘             → 齐套率/响应时间/替代率概览
```

### P2级（V2再考虑）

```
P2-1  交期预测（ML模型）
P2-2  批次/效期管理
P2-3  供应商绩效评分
P2-4  国产替代导入进度管理
P2-5  多级审批流
```

## 四、演示时的话术（功能缺口怎么"圆"）

### 对于已实现的功能

> 直接展示，不用说太多。数字说明一切。

### 对于未实现的功能

| 缺口 | 演示话术 |
|:----|:---------|
| **没有定时器** | "今天演示的是按需检查。正式部署时，Agent可以设置'每2小时自动检查'，就像一个7x24小时的AI员工盯着你的物料。" |
| **没有推送通知** | "正式版会整合企业微信/邮件推送，缺料发现后直接推送到相关负责人手机上。" |
| **没有国产替代标签** | "Agent的替代推荐引擎已经支持按供应商维度评估。正式对接SMIC数据后，可以一键筛选国产替代选项。" |
| **没有KPI面板** | "今天的重点看Single Agent的决策能力。KPI仪表盘是我们V1的标准功能，今天先看核心能力。" |

Day 1:

Week 1:

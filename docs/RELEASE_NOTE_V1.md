# 智衍 EvolvIQ · V1 版本发布说明（Release Note）

> 版本：V1（战略增强版）　发布日期：2026-07-16　适用平台：Agent Studio MVP
> 编制：工业5点0产业生态联盟 · 智衍 EvolvIQ 团队

## 一、版本定位

V1 在既有 MVP（11 Agent 真实落地 · LLM 推理层接通 · 数据层落库 · 供应链 ROI 闭环）基础上，闭合了 02 方案 P2 清单中挂起的 **4 个战略项**，使平台在「知识沉淀、能力开放、协议接入、策略自治」四个维度达到生产可用：

| 战略项 | 能力 | 交付形态 |
|--------|------|----------|
| **V1-1** | 跨 Agent 知识图谱 | Neo4j + 内存图双模 · `/kg/*` API · 前端知识图谱 Tab |
| **V1-2** | 全 11 Agent MCP 能力层联邦 | 统一联邦注册表（38 工具）· HTTP + stdio 双传输 |
| **V1-3** | 工业协议网关齐备 | Modbus / MQTT / OPC-UA / IPC-CFX 四类 · `/gateways` API |
| **V1-4** | 控制台「按效果调参」 | 策略调参引擎 · `/strategy` API · 前端策略调参 Tab |

## 二、核心交付与实证

### V1-1 · 跨 Agent 知识图谱
- **能力**：把 11 个 Agent 的工业语义（物料 / 设备 / 产品 / 缺陷 / 产线 / 部件 / 订单 / 元件 / 标准 / 变更 / 良率）桥接成一张跨域语义网；质量案例经设备类型关键词自动桥接 PM 设备与部件；引擎执行结果以 fire-and-forget 方式增量写入图谱。
- **实证**：种子构建 **102 节点 / 95 边**；Neo4j 不可达时自动回退内存图（dict 邻接表），生产可达自动切换 Neo4j 模式，**零启动中断**。
- **前端**：知识图谱 Tab（统计 + 按类型/邻居查询 + 跨 Agent 桥接示例 + 力导向图可视化）。

### V1-2 · 全 11 Agent MCP 能力层联邦
- **决策**：采用「能力层联邦」——统一 MCP server 对外暴露 11 Agent 的 in-process 工具，Agent 内部仍走 in-process 主路径，**与 T4「收敛删死代码」方向一致，零运行时风险**（未采用执行层联邦，避免反向返工）。
- **能力**：`{agent}__{method}` 命名空间统一注册 **38 个工具**（供应链 6 / 设备 4 / 良率 4 / 其余 8 Agent 各 3）；HTTP（`mcp_tools.py`）+ stdio（`mcp_server.py`）双传输均从联邦注册表驱动；保留 6 个旧版供应链工具短名别名以兼容既有 MCP 客户端。
- **实证**：11/11 Agent 代表工具经真实 API 全 200；未知工具正确 404；集成测试 `/mcp/tools` 6 工具契约保持不变。

### V1-3 · 工业协议网关齐备
- **能力**：四类工业协议网关（Modbus / MQTT / OPC-UA / IPC-CFX）均实现 `BaseGateway` 统一接口（read / write / connect / health）；`GatewayManager` 统一持有并 best-effort 初始化；`/gateways` API 暴露健康总览、单网关详情与实时读数。
- **实证**：沙箱无真实 OPC-UA Server / AMQP Broker 时，四类网关自动 `simulated` 模式，读数正常、绝不破管；**附带收益**——Modbus / MQTT 经 `GatewayManager` 首次获得运行时可观测能力，消除原死代码隐患。
- **前端**：网关 Tab（4 类网关健康卡 + 实时读数轮询面板）。

### V1-4 · 控制台「按效果调参」
- **能力**：`StrategyTuner` 汇聚「放权度（自主率）vs 人类信任度（介入批准率/驳回）」，生成调参建议（放权 / 收紧）；采纳建议或手动调参后**实时改写运行时授权边界**并留审计轨迹。
- **关键修复**：补齐漏掉的 `yield_analysis` 第 11 个授权边界，使 11/11 Agent 均有授权边界。
- **实证**：supply_chain 置信阈值 `0.8 → 0.72` **运行时真正改写**；调参夹紧区间 [0.5, 0.95]；非法参数 400 / 未知 Agent 404；审计轨迹可查。
- **铁律**：调参引擎只调整策略阈值，绝不改写任何业务数字或动作。
- **前端**：策略调参 Tab（旋钮 + 效果一页视图、建议采纳、手动调参、审计轨迹）。

## 四、架构与韧性说明（升级要点）

1. **韧性降级铁律**：所有外部依赖（PostgreSQL / Neo4j / OPC-UA Server / AMQP Broker / MCP 客户端库）均**惰性导入 + 不可达自动回退本地替代**（SQLite / 内存图 / simulated），绝不阻断启动或执行管道。
2. **幂等自初始化**：需 FastAPI lifespan 初始化的组件（网关管理器）在 API handler 内以 `ensure_ready()` 幂等守卫首调自初始化，兼容「寿命周期未跑」的调用场景。
3. **事实锚点铁律**：调参 / 优化只改策略阈值或实体关系，绝不改写业务数字或动作。
4. **收敛一致**：V1-2 能力层联邦与 T4「收敛删死代码」方向一致，未新增 Agent 执行分支。

## 五、剩余范围（非本次发布）

- **多 Agent 协作 / 编排**：与 T4「收敛」方向相反，为既定非目标，V1 未纳入。
- 多租户 / 企业版隔离（V2 范围）：暂无 `tenant_id` / `plant_id` 三级隔离。
